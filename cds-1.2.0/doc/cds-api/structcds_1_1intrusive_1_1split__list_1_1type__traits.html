typedef CDS_DEFAULT_ALLOCATOR cds::intrusive::split_list::type_traits::allocator

Allocator for bucket table. Default is CDS_DEFAULT_ALLOCATOR

typedef cds::backoff::Default cds::intrusive::split_list::type_traits::back_off

back-off strategy used

If the option is not specified, the cds::backoff::Default is used.

typedef opt::none cds::intrusive::split_list::type_traits::hash

Hash function.

Hash function converts the key fields of struct T stored in the split list into value of type size_t called hash value that is an index of hash table.

This is mandatory type and has no predefined one.

typedef atomicity::item_counter cds::intrusive::split_list::type_traits::item_counter

Item counter.

The item counting is an important part of SplitListSet algorithm: the empty() member function depends on correct item counting. Therefore, atomicity::empty_item_counter is not allowed as a type of the option.

Default is atomicity::item_counter.

typedef opt::v::relaxed_ordering cds::intrusive::split_list::type_traits::memory_model

C++ memory model for atomic operations.

Can be opt::v::relaxed_ordering (relaxed memory model, the default) or opt::v::sequential_consistent (sequentially consisnent memory model).

const bool cds::intrusive::split_list::type_traits::dynamic_bucket_table = true
static

What type of bucket table is used.

true - use split_list::expandable_bucket_table that can be expanded if the load factor of the set is exhausted false - use split_list::static_bucket_table that cannot be expanded

Default is true.
